# ZenitarParser Pro — Telegram Parser · Inviter · Sender

**ZenitarParser Pro** — production-ready инструмент для рекламы и работы с аудиторией в Telegram. Парсинг участников и активных пользователей, инвайтинг и массовая рассылка с мульти-аккаунтной ротацией и защитой от банов. Полностью управляется через Telegram-бота.

---

## 🚀 Возможности

### 🔍 Парсер
- **Участники группы** — все / недавно активные / администраторы / только боты
- **Активные пользователи** — кто реально пишет в чате (по истории сообщений)
- **Поиск чатов по ключевым словам** — находит публичные группы и каналы
- **Парсинг по реакциям** — собирает тех, кто поставил реакцию на пост
- **Парсинг комментаторов** — кто комментировал пост канала
- Экспорт в CSV с полными данными (id, username, имя, телефон, premium, verified)
- История экспортов прямо в боте

### 🎯 Аудитория (операции над списками)
- **Дедупликация** — убрать повторы (по id или username)
- **Объединение** нескольких CSV в один
- **Вычитание** — исключить из списка A всех из списка B (например, уже приглашённых)
- **Фильтры** — только с @username / только Premium / без ботов / только живые люди

### 📨 Инвайтер
- Массовое добавление пользователей из CSV в группу
- **Мульти-аккаунтная ротация** — работа распределяется по всем аккаунтам
- Защита от `PeerFlood`: аккаунт автоматически уходит на кулдаун
- Дневные лимиты на каждый аккаунт (анти-бан)
- Случайные задержки между действиями

### 📢 Рассыльщик
- **Через юзербот (Pyrogram)** — от имени реальных аккаунтов, ротация по пулу
- **Через бота (Bot API)** — массово и быстро, ротация по нескольким ботам
- Шаблоны с переменными: `{name}` `{username}` `{last_name}` `{full_name}`
- **Превью и тест-отправка себе** перед запуском массовой рассылки

### 🛡 Production-функции
- Мульти-аккаунт с трекингом здоровья (активен / кулдаун / бан / лимит)
- Поддержка **SOCKS5/HTTP прокси** на каждый аккаунт
- Добавление аккаунтов прямо в боте (код подтверждения + 2FA)
- SQLite-база (настройки, токены, лимиты, статистика)
- Статистика и история операций
- Логирование с ротацией файлов
- Опциональный персистентный FSM через Redis
- Глобальная обработка ошибок, graceful shutdown
- Docker / docker-compose

## 📦 Установка

### Вариант 1 — Docker (рекомендуется для прода)

```bash
git clone https://github.com/SyberianIT/ZenitarParser_bot-Python-Telegram.git
cd ZenitarParser_bot-Python-Telegram/ZenitarParser

cp .env.example .env
# отредактируйте .env (см. ниже)

docker compose up -d --build
docker compose logs -f bot
```

### Вариант 2 — локально

```bash
cd ZenitarParser_bot-Python-Telegram/ZenitarParser
./setup.sh                 # создаёт venv, ставит зависимости, генерирует .env
# отредактируйте .env
source venv/bin/activate
python main.py
```

## ⚙ Настройка (`.env`)

```ini
API_ID=12345                  # https://my.telegram.org
API_HASH=ваш_api_hash         # https://my.telegram.org
BOT_TOKEN=токен_бота          # @BotFather
ADMIN_IDS=123456789           # ваш Telegram ID (несколько через запятую)

MAX_INVITES_PER_DAY=40        # лимит инвайтов на аккаунт в сутки
MAX_MESSAGES_PER_DAY=30       # лимит сообщений на аккаунт в сутки
FLOOD_COOLDOWN=3600           # кулдаун после флуда (сек)

DEFAULT_DELAY_INVITE=20-45    # задержка инвайтинга, мин-макс сек
DEFAULT_DELAY_SEND=15-40      # задержка рассылки, мин-макс сек

# REDIS_URL=redis://redis:6379/0   # опционально, для персистентного FSM
LOG_LEVEL=INFO
```

| Параметр | Где взять |
|----------|-----------|
| `API_ID`, `API_HASH` | https://my.telegram.org → API development tools |
| `BOT_TOKEN` | [@BotFather](https://t.me/BotFather) |
| `ADMIN_IDS` | [@userinfobot](https://t.me/userinfobot) |

## 📖 Использование

1. `/start` — открыть панель управления
2. **👥 Аккаунты → ➕ Добавить аккаунт** — добавьте хотя бы один аккаунт
   (опционально прокси → номер телефона → код из Telegram → 2FA при наличии)
3. **🔍 Парсер** — выберите тип, получите CSV
4. **📨 Инвайтер** — загрузите CSV (или возьмите последний парс) → укажите группу
5. **📢 Рассыльщик** — выберите режим → загрузите получателей → введите текст
6. **📊 Статистика** — сводка по всем операциям
